Comments within SQL SQL Server

В этом учебном пособии вы узнаете, как использовать комментарии в ваших SQL операторах в SQL Server (Transact-SQL) с синтаксисом и примерами.

Описание

Комментарии в SQL-операторах в SQL Server (Transact-SQL) могут отображаться в одной строке или в нескольких строках.

Синтаксис

Существует два вида синтаксиса, которые можно использовать для создания комментария в SQL-выражении в SQL Server (Transact-SQL).

Синтаксис

Синтаксис использования символов -- (два минуса)

Синтаксис создания комментария SQL с использованием -- символов в SQL Server (Transact-SQL):

-- здесь находится комментарий

В SQL Server комментарий, начинающийся с символов --, должен находиться в конце строки в вашем SQL. Этот метод комментирования может охватывать только одну строку внутри вашего SQL и должен находиться в конце строки.

Синтаксис Использование /* и */ символов

Синтаксис создания комментария SQL с использованием /* и */ символов в SQL Server (Transact-SQL):

/* здесь находится комментарий */

В SQL Server комментарий, который начинается с символа /* и заканчивается на */, может быть в любом месте вашего оператора SQL. Этот метод комментирования может охватывать несколько строк в вашем SQL.

Пример - комментарий к отдельной строке

Вы можете создать комментарий SQL в одной строке в своем SQL-выражении в SQL Server (Transact-SQL).

Рассмотрим пример SQL-комментариев, который показывает комментарий SQL в его собственной строке:

Transact-SQL
1
2
3
SELECT count(department)
/* количество отделов */
FROM products;

Вот комментарий SQL, который появляется в середине строки:

Transact-SQL
1
2
SELECT /* количество отделов */ count(department)
FROM products;

Вот комментарий SQL, который появляется в конце строки:

Transact-SQL
1
2
SELECT count(department) /* количество отделов */
FROM products;

или

Transact-SQL
1
2
SELECT count(department) -- количество отделов
FROM products;

Пример - комментарий к нескольким строкам

В SQL Server (Transact-SQL) вы можете создать комментарий, который охватывает несколько строк в вашем SQL-предложении. Например:

Transact-SQL
1
2
3
4
5
6
SELECT employee_id, last_name
/*
* Author: Паша
* Purpose: комментарий охватывает несколько строк в вашем SQL предложении.
*/
FROM employees;

В этом примере комментарий охватывает 4 строки.
