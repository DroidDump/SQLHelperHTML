MariaDB AFTER DELETE триггер

В этом учебном пособии вы узнаете, как создать триггер AFTER DELETE в MariaDB с синтаксисом и примерами.

Описание

Триггер AFTER DELETE означает, что в MariaDB сработает этот триггер после выполнения операции DELETE.

Синтаксис

Синтаксис создания триггера AFTER DELETE в MariaDB:

CREATE TRIGGER trigger_name AFTER DELETE
ON table_name FOR EACH ROW

BEGIN

-- variable declarations

-- trigger code

END;

Параметры или аргументы

trigger_name
Название создаваемого триггера.
AFTER DELETE
Это означает, что триггер сработает после выполнения операции DELET
table_name
Имя таблицы, для которой создан триггер.

Ограничения

  • Вы не можете создать триггер AFTER для VIEW.
  • Вы не можете обновить NEW значения.
  • Вы не можете обновить OLD значения.

Примечание

  • Смотрите также, как создавать триггеры AFTER INSERT, AFTER UPDATE, BEFORE DELETE, BEFORE INSERT и BEFORE UPDATE.
  • Смотрите также, как удалить триггер.

Пример

Рассмотрим пример того, как создать триггер AFTER DELETE с помощью оператора CREATE TRIGGER в MariaDB.
Если у вас есть таблица, созданная следующим образом:

CREATE TRIGGER contacts_after_delete
AFTER DELETE
   ON contacts FOR EACH ROW
 
BEGIN
 
   DECLARE vUser varchar(50);
 
   -- Поск username пользователя, выполняющего delete в таблице  
   SELECT USER() INTO vUser;
 
   -- Вставка записи в таблицу audit
   INSERT INTO contacts_audit
   ( contact_id,
     deleted_date,
     deleted_by)
   VALUES
   ( OLD.contact_id,
     SYSDATE(),
     vUser );
 
END; //
 
DELIMITER ;
