Комментарии SQL SQLite

В этом учебном пособии вы узнаете, как использовать комментарии SQL операторов в SQLite с синтаксисом и примерами.

Описание

Эти комментарии могут появляться в одной строке или охватывать несколько строк. Давайте посмотрим, как это сделать.

Синтаксис

Существует два синтаксиса, которые вы можете использовать для создания комментария в вашем выражении SQL в SQLite.

Использование синтаксиса -- символ

Синтаксис для создания комментария SQL в SQLite с использованием символа --:

-- это комментарий

В SQLite комментарий, начинающийся с символа --, похож на комментарий, начинающийся с символа #. При использовании символа - комментарий должен находиться в конце строки в вашем операторе SQL с разрывом строки после него. Этот метод комментирования может занимать только одну строку в вашем SQL и должен находиться в конце строки.

Синтаксис Использование символов /* и */

Синтаксис для создания комментария SQL в SQLite с использованием символов /* и */:

/* это комментарий */

В SQLite комментарий, который начинается с символа /* и заканчивается */ и может находиться в любом месте вашего оператора SQL. Этот метод комментирования может занимать несколько строк в вашем SQL.

Пример - комментарий в одной строке

PgSQL
1
2
3
SELECT employee_id, last_name, first_name
/* Author: Ivanov I.I.*/
FROM employees;

Вот комментарий SQL, который появляется в середине строки:

PgSQL
1
2
SELECT /* Author: Ivanov I.I.*/ employee_id, last_name, first_name
FROM employees;

Вот комментарий SQL, который появляется в конце строки:

PgSQL
1
2
SELECT employee_id, last_name, first_name /* Author: Ivanov I.I.*/
FROM employees;

ИЛИ

PgSQL
1
2
SELECT employee_id, last_name, first_name  -- Author: Ivanov I.I.
FROM employees;

Пример - комментарий к нескольким строкам

В SQLite вы можете создать комментарий SQL, который занимает несколько строк в вашем операторе SQL. Например:

PgSQL
1
2
3
4
5
6
7
SELECT employee_id, last_name, first_name
/*
* Author: Ivanov I.I.
* Purpose: Чтобы показать комментарий, который занимает несколько строк в вашем SQL
* запросе в SQLite.
*/
FROM employees;

Этот комментарий SQL охватывает несколько строк в SQLite - в этом примере он занимает 5 строк.

В SQLite вы также можете создать комментарий SQL, который занимает несколько строк, используя этот синтаксис:

PgSQL
1
2
3
SELECT employee_id, last_name, first_name /* Author: Ivanov I.I.
* Purpose: Чтобы показать комментарий, который занимает несколько строк в вашем SQL запросе в SQLite. */
FROM employees;

SQLite будет предполагать, что всё после символа /* является комментарием, пока не достигнет символа */, даже если он занимает несколько строк в операторе SQL. Таким образом, в этом примере комментарий SQL будет занимать 2 строки внутри оператора SQL.
